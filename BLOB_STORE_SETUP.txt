================================================================================
                            SETUP STEPS
================================================================================

STEP 1: Create Vercel Blob Store
---------------------------------
1. Go to: https://vercel.com/dashboard
2. Click on your project (aiyyooo)
3. Click "Storage" tab
4. Click "Create Database"
5. Select "Blob" (Blob Store)
6. Name it: ceyora-blob
7. Click "Create"

STEP 2: Connect to Your Project
--------------------------------
1. After creating, click "Connect to Project"
2. Select your project from dropdown
3. Click "Connect"
4. Vercel will automatically add environment variables:
   - BLOB_READ_WRITE_TOKEN

STEP 3: (Optional) Create KV Store for Reviews
-----------------------------------------------
If you want reviews to persist permanently:

1. In the same "Storage" tab
2. Click "Create Database" again
3. Select "KV" (Key-Value Store)
4. Name it: ceyora-kv
5. Click "Create"
6. Click "Connect to Project"
7. Select your project
8. Click "Connect"

Note: If you skip this step, reviews will work but only persist during 
the current deployment (they'll reset on redeploy).

STEP 4: Push Updated Code to GitHub
------------------------------------
git add .
git commit -m "Updated to use Vercel Blob Store"
git push

STEP 5: Redeploy to Vercel
---------------------------
If connected to GitHub, Vercel will auto-deploy.
Or manually run:

vercel --prod

================================================================================
                        BLOB STORE VS KV COMPARISON
================================================================================

BLOB STORE (For Images):
âœ“ Better for large files (images, videos)
âœ“ No size limit per file
âœ“ Faster delivery via CDN
âœ“ Automatic optimization
âœ“ Public URLs for images
âœ“ Free tier: 1 GB storage, 100 GB bandwidth

KV STORE (For Reviews - Optional):
âœ“ Better for small data (text, JSON)
âœ“ Fast key-value lookups
âœ“ Good for metadata
âœ“ Free tier: 256 MB storage
âœ“ If not available, uses memory fallback

================================================================================

KV Store (Optional):

================================================================================
                        TROUBLESHOOTING
================================================================================

Problem: Still getting environment variable error
Solution: 
1. Make sure you created Blob Store
2. Make sure you clicked "Connect to Project"
3. Redeploy: vercel --prod

Problem: Images not persisting
Solution:
1. Check Vercel dashboard â†’ Storage
2. Verify Blob Store is connected
3. Check deployment logs: vercel logs

Problem: Reviews not persisting after redeploy
Solution:
1. This is normal if you didn't create KV Store
2. Create KV Store for permanent review storage
3. Or reviews will work but reset on each deployment

Problem: Upload fails
Solution:

2. Check Vercel logs for errors
3. Verify Blob Store has available space
